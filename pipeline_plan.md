## 0. Critical Review of the Existing `report.qmd`

Before building the pipeline, it's important to be honest about what the current draft gets right and where it has significant problems.

### What works
- The overall framework (accessibility → biodiversity → spatial justice → MCDA) is logically structured.
- The 300 m buffer maps are a reasonable starting point for visualising accessibility.
- The use of `ineq::Gini()` for the green space Gini coefficient is methodologically appropriate.
- The QMD structure separates concerns (analysis sections map to SQs).

### Significant problems to fix

**1. Hardcoded absolute paths — the whole pipeline breaks on any other machine.**
Every `read_sf()` call uses `/Users/belinaileen/Desktop/...`. This is the single biggest practical issue. The pipeline needs a `config.R` or a `here::here()` root-relative approach.

**2. Wrong layer names for the data you actually have.**
The QMD loads `guangzhou_osm_green_proj` from a Guangzhou-wide dataset and clips it to Yuexiu — but according to your READMEs, you *already have* the Yuexiu-clipped, projected layers (e.g. `Yuexiu_osm_green_proj`). The clipping step is unnecessary and error-prone (double-clipping already-clipped data creates tiny slivers).

**3. The Gini calculation is spatially wrong.**
```r
# CURRENT (wrong) — uses a 50x50m grid, not the neighbourhood units
grid <- st_make_grid(yuexiu_bnd, cellsize = c(50, 50)) %>% st_sf()
green_gini <- Gini(grid_map$green_area)
```
A Gini on 50 m cells measures raw spatial fragmentation, not distributional equity across residents. The Gini should be calculated on **green space per capita per Jiēdào / Wijk**, which is the policy-relevant unit.

**4. The Delft Gini code references the wrong column.**
```r
green_gini_delft <- Gini(grid_map_delft$delft_green)  # column doesn't exist
```
`st_interpolate_aw()` produces a column called `green_area`, not `delft_green`. This will throw an error.

**5. Walkable access analysis conflates two different things.**
The "walkable green access zones" code intersects road buffers with green buffers. This is a road-proximity proxy, not actual network-based walkability. It tells you roads are near parks — not that residents can walk to parks. This should be replaced with population-weighted accessibility (what % of WorldPop pixels are within X metres of a green polygon).

**6. No subdistrict join anywhere.**
Despite the spatial units (Jiēdào for Yuexiu, Wijken for Delft) being central to the research design, the current code never actually joins any indicator to those units. Every analysis is either city-wide or grid-based. The whole comparative framework depends on these joins.

**7. No connectivity or biodiversity analysis exists yet.**
Results sections 2 and 3 are entirely placeholder text. The pipeline needs to be built from scratch for those.

**8. `install.packages()` inside code chunks.**
This causes re-installation every time the document renders and will fail in read-only environments. Move to a single setup chunk with `if (!requireNamespace(...))`.

This replaces and substantially improves the buffer code in the current QMD.

```r
source("R/00_config.R")
library(sf); library(terra); library(exactextractr)
library(dplyr); library(ggplot2)

d <- readRDS(file.path(OUT_ROOT, "yuexiu_data.rds"))
dl <- readRDS(file.path(OUT_ROOT, "delft_data.rds"))

# ── Helper: population-weighted % within buffer ─────────────────────────────
pct_pop_within_buffer <- function(green_sf, pop_rast, boundary_sf, dist_m) {
  buf <- st_union(st_buffer(green_sf, dist_m))
  pop_total  <- exact_extract(pop_rast, boundary_sf, "sum")
  pop_within <- exact_extract(pop_rast, st_intersection(boundary_sf, buf), "sum")
  pop_within / pop_total * 100
}

# ── 1A. Green space per capita per subdistrict ───────────────────────────────
calc_green_per_capita <- function(admin_sf, green_sf, pop_rast, id_col) {
  admin_sf |>
    mutate(
      green_area_m2 = as.numeric(st_area(st_intersection(geometry, st_union(green_sf)))),
      pop_count     = exact_extract(pop_rast, geometry, "sum"),
      green_pc_m2   = green_area_m2 / pop_count
    )
}

yx_sub_access <- calc_green_per_capita(d$yx_sub, d$yx_grn, d$yx_pop, "name")
dl_wijk_access <- calc_green_per_capita(dl$dl_wijk, dl$dl_grn, dl$dl_pop, "wijknaam")

# ── 1B. Population within 300 m and 500 m of any green space ─────────────────
# This is the correct population-based approach, not road-intersection proxy
yx_300 <- pct_pop_within_buffer(d$yx_grn, d$yx_pop, d$yx_bnd, 300)
yx_500 <- pct_pop_within_buffer(d$yx_grn, d$yx_pop, d$yx_bnd, 500)
dl_300 <- pct_pop_within_buffer(dl$dl_grn, dl$dl_pop, dl$dl_bnd, 300)
dl_500 <- pct_pop_within_buffer(dl$dl_grn, dl$dl_pop, dl$dl_bnd, 500)

# ── 1C. Mean nearest park distance per subdistrict ───────────────────────────
# Use st_nearest_feature + st_distance rather than centroid-only approach
nearest_park_dist <- function(admin_sf, green_sf) {
  centroids <- st_centroid(admin_sf)
  nearest_idx <- st_nearest_feature(centroids, green_sf)
  dists <- st_distance(centroids, green_sf[nearest_idx, ], by_element = TRUE)
  admin_sf |> mutate(mean_dist_to_green_m = as.numeric(dists))
}

yx_sub_access <- nearest_park_dist(yx_sub_access, d$yx_grn)
dl_wijk_access <- nearest_park_dist(dl_wijk_access, dl$dl_grn)

# ── Save outputs ─────────────────────────────────────────────────────────────
saveRDS(yx_sub_access,  file.path(OUT_ROOT, "yx_sub_access.rds"))
saveRDS(dl_wijk_access, file.path(OUT_ROOT, "dl_wijk_access.rds"))

# ── Figures ──────────────────────────────────────────────────────────────────
p1 <- ggplot(yx_sub_access) +
  geom_sf(aes(fill = green_pc_m2)) +
  scale_fill_viridis_c(name = "m² per person", option = "G") +
  theme_minimal() +
  labs(title = "Green Space per Capita — Yuexiu Jiēdào")
```

```r
source("R/00_config.R")
library(sf); library(terra); library(exactextractr)
library(dplyr); library(ggplot2)

d  <- readRDS(file.path(OUT_ROOT, "yuexiu_data.rds"))
dl <- readRDS(file.path(OUT_ROOT, "delft_data.rds"))

# ── 2A. Green space typology from OSM tags ───────────────────────────────────
# OSM green polygons carry landuse/leisure attributes — classify them
classify_green_type <- function(green_sf) {
  green_sf |>
    mutate(green_type = case_when(
      grepl("park",         leisure, ignore.case = TRUE) ~ "Park",
      grepl("garden",       leisure, ignore.case = TRUE) ~ "Garden",
      grepl("nature_reserve|forest", landuse, ignore.case = TRUE) ~ "Nature/Forest",
      grepl("grass|meadow", landuse, ignore.case = TRUE) ~ "Grass/Meadow",
      grepl("cemetery",     landuse, ignore.case = TRUE) ~ "Cemetery",
      TRUE ~ "Other green"
    ))
}
# Note: check actual column names in your .gpkg with names(d$yx_grn) first

# ── 2B. NDVI zonal statistics per green patch ────────────────────────────────
yx_grn_ndvi <- d$yx_grn |>
  mutate(ndvi_mean = exact_extract(d$yx_ndvi, geometry, "mean"),
         ndvi_sd   = exact_extract(d$yx_ndvi, geometry, "stdev"))

dl_grn_ndvi <- dl$dl_grn |>
  mutate(ndvi_mean = exact_extract(dl$dl_ndvi, geometry, "mean"),
         ndvi_sd   = exact_extract(dl$dl_ndvi, geometry, "stdev"))

# ── 2C. GBIF species density per green patch (species per hectare) ───────────
species_density <- function(green_sf, gbif_sf) {
  counts <- lengths(st_intersects(green_sf, gbif_sf))
  green_sf |>
    mutate(gbif_count    = counts,
           species_per_ha = gbif_count / area_ha)
}

yx_grn_bio <- species_density(yx_grn_ndvi, d$yx_gbif)
dl_grn_bio <- species_density(dl_grn_ndvi, dl$dl_gbif)

# ── 2D. Blue-green ratio per subdistrict ─────────────────────────────────────
# Use ESA WorldCover: class 20 = shrubland, 30 = grassland, 40 = cropland,
#                      10 = tree cover, 80 = permanent water
bg_ratio <- function(admin_sf, cover_rast) {
  admin_sf |>
    mutate(
      pct_green = exact_extract(cover_rast, geometry,
        fun = function(values, cov) mean(values %in% c(10, 20, 30, 40), na.rm = TRUE) * 100),
      pct_water = exact_extract(cover_rast, geometry,
        fun = function(values, cov) mean(values == 80, na.rm = TRUE) * 100),
      bg_ratio  = pct_green / (pct_water + 0.001)
    )
}

# Save
saveRDS(yx_grn_bio, file.path(OUT_ROOT, "yx_grn_bio.rds"))
saveRDS(dl_grn_bio, file.path(OUT_ROOT, "dl_grn_bio.rds"))
```

### `05_connectivity.R` — SQ4

This is the most technically demanding script and should be owned by one person.

```r
source("R/00_config.R")
library(sf); library(igraph); library(dplyr)
library(landscapemetrics); library(terra); library(ggplot2)

d  <- readRDS(file.path(OUT_ROOT, "yuexiu_data.rds"))
dl <- readRDS(file.path(OUT_ROOT, "delft_data.rds"))

# ── 4A. Landscape fragmentation metrics (landscapemetrics) ───────────────────
# Requires binary raster: green = 1, non-green = 0
# Use the WorldCover raster, reclassify tree/grass/shrub as green
make_green_raster <- function(cover_rast) {
  m <- matrix(c(10, 1,   # tree cover
                20, 1,   # shrubland
                30, 1,   # grassland
                40, 0,   # cropland
                50, 0,   # built-up
                60, 0,   # bare
                80, 0,   # water
                95, 1),  # mangrove
              ncol = 2, byrow = TRUE)
  classify(cover_rast, m, others = 0)
}

yx_green_rast <- make_green_raster(d$yx_cover)
dl_green_rast <- make_green_raster(dl$dl_cover)

# Core metrics: number of patches (NP), mean patch size (MPS),
#               euclidean nearest-neighbour distance (ENN)
yx_frag <- calculate_lsm(yx_green_rast, what = c("lsm_c_np", "lsm_c_area_mn", "lsm_c_enn_mn"))
dl_frag <- calculate_lsm(dl_green_rast, what = c("lsm_c_np", "lsm_c_area_mn", "lsm_c_enn_mn"))

# ── 4B. Functional connectivity graph ────────────────────────────────────────
# Nodes = green patch centroids; edges where centroids are within dispersal threshold
build_connectivity_graph <- function(green_sf, threshold_m) {
  cents <- st_centroid(green_sf)
  dist_mat <- st_distance(cents)          # n × n distance matrix

  edges <- which(dist_mat > units::as_units(0, "m") &
                   dist_mat <= units::as_units(threshold_m, "m"),
                 arr.ind = TRUE)

  g <- graph_from_edgelist(edges, directed = FALSE)
  V(g)$patch_id   <- seq_len(nrow(green_sf))
  V(g)$area_ha    <- green_sf$area_ha
  E(g)$dist_m     <- as.numeric(dist_mat[edges])

  # Betweenness centrality — identifies stepping-stone patches
  V(g)$betweenness <- betweenness(g, normalized = TRUE)
  g
}

yx_graph <- build_connectivity_graph(d$yx_grn,  DISPERSAL_THRESH_M)
dl_graph <- build_connectivity_graph(dl$dl_grn, DISPERSAL_THRESH_M)

# Key metrics
cat("Yuexiu — components:", components(yx_graph)$no,
    "| avg betweenness:", mean(V(yx_graph)$betweenness), "\n")
cat("Delft  — components:", components(dl_graph)$no,
    "| avg betweenness:", mean(V(dl_graph)$betweenness), "\n")

# Add betweenness back to sf for mapping
d$yx_grn <- d$yx_grn |>
  mutate(betweenness = V(yx_graph)$betweenness,
         component   = components(yx_graph)$membership)

saveRDS(list(yx_graph=yx_graph, dl_graph=dl_graph,
             yx_frag=yx_frag, dl_frag=dl_frag,
             yx_grn_conn=d$yx_grn),
        file.path(OUT_ROOT, "connectivity_results.rds"))
```

This is the synthesis script that combines all prior outputs.

```r
source("R/00_config.R")
library(sf); library(dplyr); library(ggplot2); library(viridis)

# Load all prior outputs
access  <- readRDS(file.path(OUT_ROOT, "yx_sub_access.rds"))
bio     <- readRDS(file.path(OUT_ROOT, "yx_grn_bio.rds"))
sj      <- readRDS(file.path(OUT_ROOT, "spatial_justice_results.rds"))
conn    <- readRDS(file.path(OUT_ROOT, "connectivity_results.rds"))

# ── Step 1: Aggregate patch-level indicators to subdistrict level ─────────────
# Join green patch biodiversity and connectivity metrics to Jiēdào
yx_sub <- access  # start from subdistrict access layer

# Mean NDVI per subdistrict (from patch means, area-weighted)
grn_bio <- bio |> st_drop_geometry()
yx_sub_bio <- yx_sub |>
  st_join(bio |> select(ndvi_mean, species_per_ha, betweenness=betweenness),
          join = st_intersects) |>
  st_drop_geometry() |>
  group_by(across(-c(ndvi_mean, species_per_ha, betweenness))) |>
  summarise(ndvi_mean      = mean(ndvi_mean, na.rm = TRUE),
            species_per_ha = mean(species_per_ha, na.rm = TRUE),
            mean_betweenness = mean(betweenness, na.rm = TRUE)) |>
  ungroup()

# Rejoin geometry
yx_mcda <- yx_sub |>
  left_join(st_drop_geometry(yx_sub_bio), by = "geometry")

# ── Step 2: Min-max normalise each indicator to [0, 1] ───────────────────────
normalise <- function(x) (x - min(x, na.rm=TRUE)) / (max(x, na.rm=TRUE) - min(x, na.rm=TRUE))

yx_mcda <- yx_mcda |>
  mutate(
    score_access  = normalise(green_pc_m2),
    score_bio     = normalise(ndvi_mean) * 0.5 + normalise(species_per_ha) * 0.5,
    score_conn    = normalise(mean_betweenness),
    # For equity: INVERT green_pc_m2 rank so LOW access = HIGH priority
    score_equity  = 1 - normalise(green_pc_m2)
  )

# ── Step 3: Weighted MCDA score ──────────────────────────────────────────────
yx_mcda <- yx_mcda |>
  mutate(
    mcda_score = MCDA_WEIGHTS["accessibility"]  * score_access  +
                 MCDA_WEIGHTS["biodiversity"]   * score_bio     +
                 MCDA_WEIGHTS["connectivity"]   * score_conn    +
                 MCDA_WEIGHTS["social_equity"]  * score_equity
  )

# ── Step 4: NBS Corridor Prioritisation ──────────────────────────────────────
# Priority = subdistricts with LOW access AND isolated green patches
# Isolation is flagged by patch belonging to a small graph component
conn_data <- conn$yx_grn_conn

# Flag isolated patches (component size < 3 patches)
comp_sizes <- table(conn_data$component)
conn_data <- conn_data |>
  mutate(isolated = comp_sizes[as.character(component)] < 3)

# Spatially join: which Jiēdào contain isolated patches?
isolated_patches <- conn_data |> filter(isolated)
yx_mcda <- yx_mcda |>
  mutate(has_isolated_patch = lengths(st_intersects(geometry, isolated_patches)) > 0)

# Priority tier: high MCDA score + has isolated patches = Tier 1 intervention
yx_mcda <- yx_mcda |>
  mutate(priority_tier = case_when(
    mcda_score > quantile(mcda_score, 0.66) & has_isolated_patch ~ "Tier 1 — Urgent",
    mcda_score > quantile(mcda_score, 0.33)                       ~ "Tier 2 — Moderate",
    TRUE                                                           ~ "Tier 3 — Monitor"
  ))

# ── Step 5: Final map ─────────────────────────────────────────────────────────
p_mcda <- ggplot(yx_mcda) +
  geom_sf(aes(fill = mcda_score)) +
  scale_fill_viridis_c(option = "inferno", name = "MCDA Score\n(higher = more urgent)") +
  theme_minimal() +
  labs(title = "MCDA Priority Score — Yuexiu Jiēdào",
       subtitle = paste0("Weights: Accessibility ",
                         MCDA_WEIGHTS["accessibility"]*100, "% | Biodiversity ",
                         MCDA_WEIGHTS["biodiversity"]*100, "% | Connectivity ",
                         MCDA_WEIGHTS["connectivity"]*100, "% | Equity ",
                         MCDA_WEIGHTS["social_equity"]*100, "%"))

p_priority <- ggplot(yx_mcda) +
  geom_sf(aes(fill = priority_tier)) +
  scale_fill_manual(values = c("Tier 1 — Urgent"   = "#d73027",
                               "Tier 2 — Moderate" = "#fee090",
                               "Tier 3 — Monitor"  = "#91cf60"),
                    name = "NBS Priority") +
  theme_minimal() +
  labs(title = "NBS Corridor Prioritisation — Yuexiu")

ggsave(file.path(OUT_ROOT, "fig_mcda_yuexiu.png"),    p_mcda,     width=10, height=8)
ggsave(file.path(OUT_ROOT, "fig_priority_yuexiu.png"), p_priority, width=10, height=8)

saveRDS(yx_mcda, file.path(OUT_ROOT, "mcda_final.rds"))
message("MCDA complete.")
```

## 7. Key Methodological Notes for the Report

### Spatial unit mismatch
Delft's Wijken (~10 units) are larger than Yuexiu's 18 Jiēdào but comparable at city level. Note this explicitly using the Modifiable Areal Unit Problem (MAUP) discussion already outlined in the QMD. Do not aggregate Delft down to Buurten (91 units) for the main comparison — the scales would diverge too much.

### Why VIIRS ≠ income
VIIRS nighttime light is a proxy for economic activity, not household income. The correlation with green space access may reflect commercial density rather than residential affluence. The Delft income–green correlation is more direct and should be flagged as methodologically cleaner.

### GBIF bias warning
Your QMD already flags this, but it's worth quantifying: run a simple map of GBIF point density overlaid on road density. If they correlate strongly, the "species density" metric is partly a sampling effort artifact, not ecology. Consider a spatial thinning step (one point per 100 m grid cell) to reduce bias.

### MCDA weight sensitivity
After finalising the MCDA, run a simple sensitivity test: vary each weight by ±10% and check how much the priority tier assignments change. If Tier 1 areas are stable across weight variations, your result is robust. If they flip, acknowledge this as a limitation and report results for two weight scenarios.

## 8. Corrected Methodology Statement (for the Report)

Replace the current vague MCDA paragraph with this:

> The MCDA integrates four normalised indicator scores per Jiēdào / Wijk: (1) accessibility, measured as green space per capita (m²/person); (2) biodiversity, as the area-weighted mean NDVI and GBIF species density per green patch within the unit; (3) connectivity, as the mean betweenness centrality of green patches based on a dispersal-threshold graph at 150 m; and (4) social equity, as the inverse rank of green space per capita (prioritising under-served units). Each indicator is min-max normalised to [0, 1] and aggregated using weights of 30 %, 25 %, 25 %, and 20 % respectively. NBS corridor priority tiers are assigned by combining the MCDA score with a flag for subdistricts containing spatially isolated green patches (graph component size < 3), identifying locations where investment in green infrastructure would simultaneously address access deficits and restore ecological connectivity.
